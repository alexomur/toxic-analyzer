# Toxic Analyzer

Toxic Analyzer — монорепозиторий будущего проекта по анализу токсичности текстовых комментариев.

## Текущее состояние

Репозиторий заранее организован как monorepo с тремя направлениями:

- `model/` — активная зона текущего этапа
- `backend/` — зарезервировано на будущее
- `frontend/` — зарезервировано на будущее

Сейчас активная разработка ведётся только в `model/`.
Папки `frontend/` и `backend/` пока не реализуются и не должны определять архитектурные решения текущего этапа.

## Model-First Workspace

Каталог `model/` используется для разработки первой воспроизводимой baseline-модели.
Целевой домен на текущем этапе — смешанный русскоязычный UGC, собранный из нескольких источников, а не `Habr-only`.

- `notebooks/` используются только для экспериментов
- итоговый код должен жить в обычных Python-модулях
- подтверждённые решения можно переносить из ноутбуков в `model/src/`
- обучение, оценка и инференс baseline-модели реализуются только внутри `model/`

## Практические правила

- Используйте `.env.example` как шаблон локальной конфигурации.
- Не коммитьте датасеты, чекпоинты, артефакты обучения и временные результаты.
- Python-tooling для model находится в `model/pyproject.toml`.
